Assignment 1 - Voronoi diagrams

Description
 In this assignment you will create a discrete Voronoi diagram using the rendering pipeline. Voronoi diagrams are one of the main tools in procedural content generation (Fun With Voronoi Graphs).

Given a set of seed points in a plane, a Voronoi diagram is created by partitioning the surface of the plane into regions, so that all points closer to a given seed point are assigned to the region of that particular seed point. The figure in the left shows a Voronoi diagram, where each region is colored using a color attribute defined by its seed point.

Your implementation will take the paper Fast Computation of Generalized Voronoi Diagrams Using Graphics Hardware, by Hoff et al., as a basis. The paper shows that, by projecting very simple 3D geometry into the 2D framebuffer, it is possible to generate discrete approximations of the Voronoi diagram very efficiently. We call it a discrete approximation because, instead of working with the infinite number of points contained in a plane, we are working with a grid of pixels, as defined by the framebuffer of our OpenGL application.

The seed points should be defined at runtime. By clicking on a screen location, a seed point should be added and the voronoi diagram should be updated.

Notably, you will be using the geometry of a 3D cone as a proxy to your 2D seed points. The occlusion and visibility computation made possible by the depth buffer (aka z-buffer) will let you use the rendering pipeline to project the 3D cones into the 2D framebuffer, generating Voronoi diagrams and distance maps, as demonstrated in this figure from AMD:

Besides creating Voronoi diagrams, you will also use the rendering pipeline to print the distance between the user defined seed points and all other points within that region. These images are known as a distance maps, and are used to generate Worley noise).

You are given a skeleton project to work with (git pull the exercises repository, folder assignments/voronoi). The folder contains a main.cpp, a shader class, and vertex and fragment shaders files that you should complete. The main.cpp file contains the basic GLFW/OpenGL initialization, a render loop, mouse and keyboard input callbacks, and empty functions that you will need to complete (search for the TODO string). Feel free to modify the structure of the code and files if you wish.
Learning Objectives
  • Develop most of an OpenGL application.
  • Create a scene with multiple triangle meshes.
  • Manage multiple VAOs in a single application.
  • Write simple vertex and fragment shaders, and switch the shader program at runtime.
  • Use simple user input and transform click positions in screen coordinates into positions in the normalized display coordinates.
  • Use the render pipeline and the depth buffer to project a 3D scene into a 2D Voronoi diagram and distance maps.
Voronoi 1.1
Create an instantiable cone geometry
  • Fill in the function instantiateCone following the instructions in the function.
  • To buid a cone, you can reuse the circle geometry you have created for exercises 1.7 (VBO) and 1.8 (VBO and EBO). The difference is that the cone should also hold meaningful 'z' coordinate information, which describes the distance of each vertex from the virtual camera in the range [-1, 1]. We want the center of the cone to be close to the camera, and the base of the cone to be far from the camera.
  • The radius of the cone should be sufficient to cover all the drawing area in the NDC xy plane (i.e. radius >= sqrt(2*2+2*2)).
  • Each cone should use a single RGB color, and an x and y position offset, you will set these using GLSL uniform variables.

  • The figure on the left illustrates several cones as seen in the xz plane (you won't see anything now since the shaders were not defined, and you will not replicate this image later since your camera is set in the xy plane).
Voronoi 1.3
Render a Voronoi diagram
  • Fill in the shader.vert shader following the instructions in the file (it should receive the vertex position for each vertex and an offset position for each object, set the gl_Position variable, and send the z-coordinate to the fragment shader).
  • Fill in the color.frag shader following the instructions in the file (it should receive the color information from the application as an uniform, and set the output color of the shader).
  • Make sure that your application is binding the buffer with vertex positions to the input variables of the vertex shader, and that you are sending color and position offset as uniform variables.
  • Now, you should be able to click on the application window and generate images similar to the one in the left.
